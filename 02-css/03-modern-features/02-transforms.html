2D Transforms: Translate (Move)

translate(), translateX(), translateY()
Original

How Translate Works

  • translate(x, y) - Move element horizontally and vertically
  • translateX(value) - Move horizontally only
  • translateY(value) - Move vertically only
  • Positive X moves right, positive Y moves down
  • Doesn't affect document flow (doesn't push other elements)
.element {
  transform: translate(50px, 30px);  /* Move right 50px, down 30px */
  transform: translateX(40px);        /* Move right 40px */
  transform: translateY(-40px);       /* Move up 40px */
}

How Scale Works

  • scale(factor) - Scale uniformly (1.5 = 150% size)
  • scale(x, y) - Different horizontal and vertical scaling
  • scaleX(factor) - Scale width only
  • scaleY(factor) - Scale height only
  • 1 = original size, > 1 = larger, < 1 = smaller
  • 0.5 = 50% size, 2 = 200% size
.button:hover {
  transform: scale(1.1);     /* Grow to 110% */
}

.shrink {
  transform: scale(0.8);     /* 80% of original size */
}

Hover to see rotation

How Rotate Works

  • rotate(angle) - Rotate clockwise (positive) or counter-clockwise (negative)
  • Use deg (degrees), rad (radians), or turn (full rotations)
  • 45deg = eighth turn, 90deg = quarter turn, 180deg = half turn
  • 1turn = 360deg = full circle
.element {
  transform: rotate(45deg);    /* Rotate 45 degrees */
  transform: rotate(-90deg);   /* Rotate counter-clockwise */
  transform: rotate(0.5turn);  /* Half rotation (180deg) */
}

How Skew Works

  • skewX(angle) - Tilt horizontally
  • skewY(angle) - Tilt vertically
  • skew(x, y) - Tilt both directions
  • Creates a parallelogram effect
  • Use sparingly - can be disorienting

Combining Transforms

Space-separate multiple transforms. They apply left-to-right (order matters!):

.card:hover {
  /* Move right & up, rotate, then scale */
  transform: translate(20px, -20px) rotate(25deg) scale(1.2);
}

/* Order matters! These produce different results: */
transform: rotate(45deg) translateX(100px);  /* Rotate, then move */
transform: translateX(100px) rotate(45deg);  /* Move, then rotate */

How Transform Origin Works

  • Sets the point around which transforms happen
  • Default is center center (50% 50%)
  • Can use keywords: top, bottom, left, right, center
  • Can use percentages or pixel values
  • Format: transform-origin: x y
.element {
  transform-origin: top left;      /* Rotate from top-left corner */
  transform-origin: bottom center; /* Rotate from bottom center */
  transform-origin: 20px 30px;     /* Specific pixel point */
  transform-origin: 75% 25%;       /* Percentage position */
}

Understanding 3D Transforms

  • rotateX() - Rotate around horizontal axis (top-to-bottom flip)
  • rotateY() - Rotate around vertical axis (left-to-right flip)
  • rotateZ() - Same as 2D rotate() - spin in plane
  • translateZ() - Move toward/away from viewer
  • rotate3d(x, y, z, angle) - Rotate around custom axis
  • Requires perspective on parent to see 3D effect
/* Parent needs perspective */
.container {
  perspective: 1000px;  /* Lower value = more dramatic 3D */
}

.element {
  transform-style: preserve-3d;  /* Enable 3D for children */
  transform: rotateY(45deg);
}

Best Practices

1. Use with Transitions

Always add transition for smooth animations:

.card {
  transition: transform 0.3s ease;
}
.card:hover {
  transform: translateY(-10px);
}

2. Transform Order Matters

Transforms apply left-to-right. Different orders = different results.

3. Hardware Acceleration

Transforms use GPU acceleration (especially translate3d and translateZ). Great for performance!

/* Force GPU acceleration for smooth animations */
.element {
  transform: translateZ(0);
  /* or */
  will-change: transform;
}

4. Don't Transform Text

Scaling text can make it blurry. Use font-size or adjust layout instead.

5. Accessibility

  • Don't use transforms for critical layout
  • Respect prefers-reduced-motion for animations
  • Ensure transformed elements remain accessible to screen readers

Summary

Key Takeaways

  • 2D Transforms: translate, scale, rotate, skew
  • 3D Transforms: rotateX/Y/Z, translateZ, perspective
  • Combine: Space-separate multiple transforms
  • Origin: Set pivot point with transform-origin
  • Performance: Hardware accelerated, great for animations
  • No Layout Impact: Transforms don't affect document flow
  • Browser Support: Excellent (IE10+)
