CSS Custom Properties

Modern CSS variables for maintainable, themeable designs

What are CSS Custom Properties?

Definition

CSS Custom Properties (also called CSS Variables) let you store and reuse values throughout your stylesheet. They're declared with -- prefix and accessed with var().

Why Use Them?

  • Reduce repetition and make code DRY
  • Easy theming and dark mode
  • Dynamic updates with JavaScript
  • Better maintainability
  • Cascade and inherit like normal CSS properties

Basic Syntax

/* Declare variables */
:root {
  --primary-color: #6366f1;
  --spacing: 1rem;
}

/* Use variables */
.button {
  background: var(--primary-color);
  padding: var(--spacing);
}

How It Works

Variables cascade! Set a default in :root, then override it in specific components. Child elements inherit the overridden value.

Implementing Dark Mode

Add a class like .dark-mode to your root element and redefine color variables. All components using those variables automatically update!

Calculations with Variables

calc() + CSS variables
Padding: calc(var(--base-size) * var(--multiplier)) = 2rem
Font-size: calc(var(--text-base) * 1.5) = 1.5rem 
.element {
  --base-size: 1rem;
  --multiplier: 2;

  padding: calc(var(--base-size) * var(--multiplier));
  margin: calc(var(--space-md) * 2);
}

Nested Variable Calculations

Base padding: var(--size-base) = 1rem
Double padding: var(--size-double) = 2rem
Quad padding: var(--size-quad) = 4rem

Fallback Values

var(--name, fallback)
This uses var(--custom-color, #6366f1)
Since --custom-color doesn't exist, it falls back to #6366f1
This defines --custom-color locally, so no fallback needed 
.button {
  /* Use brand color, or fallback to blue */
  background: var(--brand-color, #3b82f6);

  /* Chain fallbacks */
  color: var(--text-color, var(--fallback-text, black));
}

JavaScript Integration

Dynamic updates

Reading Variables

// Get computed value
const root = document.documentElement;
const primaryColor = getComputedStyle(root)
  .getPropertyValue('--primary-color');

Setting Variables

// Set variable dynamically
document.documentElement.style
  .setProperty('--primary-color', '#ff0000');

// Set on specific element
element.style.setProperty('--theme-color', '#00ff00');

Best Practices

1. Use :root for Global Variables

Define global design tokens in :root (same as html but with higher specificity).

2. Naming Conventions

  • Use descriptive names: --primary-color not --color1
  • Group related variables: --space-sm, --space-md
  • Use semantic names: --text-color not --black

3. Create Design Tokens

Define scales for spacing, typography, colors, shadows - everything that repeats in your design.

4. Scope Variables Appropriately

  • Global tokens in :root
  • Component-specific in component selector
  • Theme overrides in theme classes

Browser Support

Excellent support in all modern browsers. IE11 does not support custom properties. For IE11, use a preprocessor or PostCSS plugin to transform variables to static values.

CSS Variables vs Preprocessor Variables

Feature CSS Variables Sass/Less Variables
Scope Runtime (DOM) Compile-time only
Dynamic Yes (JS can change) No (static)
Cascade Yes (inherit) No
Media Queries Can change per query Static value
DevTools Visible & editable Compiled away

Best of Both Worlds

Use Sass for logic (loops, mixins, functions) and CSS variables for dynamic theming. They complement each other!

Summary

Key Takeaways

  • CSS variables are declared with --name and used with var(--name)
  • They cascade and inherit like normal CSS properties
  • Perfect for theming, design systems, and dynamic styling
  • Can be updated with JavaScript in real-time
  • Use var(--name, fallback) for fallback values
  • Works great with calc() for dynamic calculations
  • Excellent browser support (except IE11)

When to Use CSS Variables

  • Design systems and tokens
  • Theming (light/dark mode)
  • Component variants
  • Responsive design (changing values per breakpoint)
  • Dynamic styling with JavaScript
  • Reducing repetition in stylesheets
