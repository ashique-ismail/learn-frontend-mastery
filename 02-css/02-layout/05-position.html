Static Positioning (Default)

position: static

What is Static?

This is the default positioning for all elements. Elements appear in the normal document flow. Top, right, bottom, left, and z-index properties have no effect.

Relative Positioning

position: relative

What is Relative?

Elements are positioned relative to their normal position. The element still occupies its original space in the document flow, but visually appears offset.

Key Points

  • Original space is preserved (shown by dashed red box)
  • Uses top, right, bottom, left for offset
  • Great for small adjustments without affecting layout
  • Creates positioning context for absolute children

Absolute Positioning

position: absolute

What is Absolute?

Elements are removed from normal document flow and positioned relative to their nearest positioned ancestor (any position except static). If no positioned ancestor exists, it positions relative to the viewport.

Key Points

  • Removed from document flow (doesn't affect other elements)
  • Positioned relative to nearest positioned ancestor
  • Perfect for overlays, tooltips, dropdowns, badges
  • Can use negative values for offset

Fixed Positioning

position: fixed

What is Fixed?

Elements are positioned relative to the viewport and stay in the same place even when scrolling. Perfect for navigation bars, back-to-top buttons, and notification badges.

Look at the bottom-right corner! There's a fixed button that stays in place as you scroll.

Scroll down on this page to see the fixed header appear at the top.

Fixed positioning is perfect for:

  • Navigation bars
  • Chat widgets
  • Cookie consent banners
  • Back to top buttons
  • Shopping cart icons

Key Points

  • Always positioned relative to viewport
  • Stays in place during scrolling
  • Removed from normal document flow
  • Often combined with high z-index

Z-Index (Stacking Order)

z-index property

What is Z-Index?

Controls the stacking order of positioned elements. Higher values appear in front. Only works on positioned elements (not static).

z-index: 1
z-index: 2
z-index: 3
z-index: -1
(behind parent)

Stacking Context

Elements create a new stacking context when they have:

  • Position + z-index (not auto)
  • Opacity < 1
  • Transform, filter, perspective
  • Flex/grid items with z-index

Children of a stacking context can't appear between elements of another context, no matter the z-index.

Z-Index Value Common Use
-1 Behind parent background
0-10 Normal content layers
100-999 Dropdowns, tooltips
1000-9999 Modals, overlays
10000+ Critical UI (notifications)

Position Properties Comparison

Property Positioned Relative To Occupies Space Scrolls Common Use Cases
static Normal flow Yes Yes Default, normal content
relative Self Yes Yes Small adjustments, positioning context
absolute Nearest positioned ancestor No With container Overlays, tooltips, dropdowns
fixed Viewport No No Headers, buttons, notifications
sticky Container + viewport Yes Until threshold Table headers, sidebars

Best Practices & Tips

1. Positioning Context

When using absolute positioning, wrap it in a relative container:

.container { position: relative; }
.absolute-child { position: absolute; top: 0; right: 0; }

2. Z-Index Strategy

Create a z-index scale and stick to it:

  • Use increments (100, 200, 300) not (1, 2, 3)
  • Document your z-index layers
  • Avoid z-index: 99999

3. Accessibility

  • Fixed elements should not obscure content
  • Ensure keyboard navigation works with positioned elements
  • Use aria-hidden for decorative positioned elements

4. Performance

  • Fixed and sticky can cause repaint issues
  • Use will-change sparingly for smooth animations
  • Minimize layers with high z-index

5. Common Pitfalls

  • Z-index not working? Element must be positioned (not static)
  • Absolute element overflow? Parent needs overflow: hidden
  • Sticky not sticking? Check parent height and overflow
  • Fixed element too wide? Set explicit width or use left+right
