Column Width Based

column-width: 200px (auto-adjusts)

With column-width, the browser automatically calculates how many columns will fit. This is more responsive than using column-count. Resize your browser to see the effect!

The browser will create as many columns as possible that are at least the specified width. This creates a naturally responsive layout without media queries.

This is particularly useful when you don't know the exact width of the container or want the layout to adapt smoothly.

Column Rules (Dividers)

column-rule: 3px solid

Column rules act like borders between columns. They don't take up space and won't push columns apart - they appear in the gap between columns.

You can style them just like borders using width, style (solid, dashed, dotted, double), and color. They're purely decorative and don't affect layout.

Perfect for creating newspaper or magazine-style layouts with visual separation between columns.

column-rule: 5px double

Here's a fancier double-line rule. Column rules can use any border style: solid, dashed, dotted, double, groove, ridge, inset, or outset.

You can set the rule using the shorthand property (column-rule) or individual properties (column-rule-width, column-rule-style, column-rule-color).

The rule will only appear if there are multiple columns. With a single column, no rule is displayed.

Column Spanning

column-span: all

Sometimes you want an element to span across all columns, like a heading or figure. That's where column-span comes in.

This Heading Spans All Columns!

After a spanning element, the content continues in columns below it. This is perfect for section headings or full-width images in a multi-column article.

Note that column-span only accepts two values: 'none' (default) or 'all'. You can't span just 2 out of 3 columns, for example.

This creates natural breaks in your content and helps organize long articles into readable sections while maintaining the multi-column flow.

Column Fill (Balance)

Balanced (default)

column-fill: balance

With balanced fill, the browser tries to distribute content evenly across all columns, making them approximately the same height.

This is the default behavior and works well for most content.

It creates a more aesthetically pleasing layout.

Auto Fill

column-fill: auto

With auto fill, columns are filled sequentially. The first column is filled completely before moving to the next.

This can result in uneven column heights.

Use this when order matters more than visual balance.

Break Properties

  • break-inside: avoid - Prevents breaks inside element
  • break-before: column - Forces break before element
  • break-after: column - Forces break after element
  • break-inside: avoid-column - Specifically for columns

Best Practices & Tips

When to Use Multi-Column

  • Long-form text: Articles, blog posts, documentation
  • Lists: Long lists that benefit from horizontal scanning
  • Gallery layouts: Card-based or masonry-style designs
  • Magazine layouts: Editorial content with images and text

Key Properties

  • column-count - Number of columns
  • column-width - Minimum width (responsive)
  • column-gap - Space between columns
  • column-rule - Border between columns
  • column-span - Span across columns (none/all)
  • column-fill - Balance or auto fill
  • break-inside - Control breaks

Shorthand Properties

  • columns: 3 200px - Sets both count and width
  • column-rule: 2px solid #333 - All rule properties

Browser Support

Excellent support across all modern browsers. Use prefixes for older browsers:

-webkit-column-count, -moz-column-count
