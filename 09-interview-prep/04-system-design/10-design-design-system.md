# Design a Design System

## What Is a Design System?

A design system is the source of truth for UI: design tokens (the values), components (the implementations), and documentation. It ensures visual consistency across a product and speeds up development by providing reusable, tested building blocks.

---

## Layer 1: Design Tokens

Tokens are named design decisions — the "atoms" of the system.

```ts
// tokens/base.ts — raw values (don't use directly in components)
export const base = {
  color: {
    blue100: '#dbeafe',
    blue500: '#3b82f6',
    blue900: '#1e3a8a',
  },
  spacing: {
    1: '4px',
    2: '8px',
    4: '16px',
    8: '32px',
  },
  fontSize: {
    sm: '14px',
    base: '16px',
    lg: '18px',
    xl: '24px',
  },
};

// tokens/semantic.ts — meaningful aliases (use these in components)
export const tokens = {
  color: {
    primary: base.color.blue500,
    primaryHover: base.color.blue900,
    background: '#ffffff',
    surface: base.color.blue100,
    text: '#111827',
    textMuted: '#6b7280',
  },
  spacing: base.spacing,
};
```

**CSS Custom Properties output (generated by Style Dictionary or manually):**
```css
:root {
  --color-primary: #3b82f6;
  --color-primary-hover: #1e3a8a;
  --color-background: #ffffff;
  --color-text: #111827;
  --spacing-1: 4px;
  --spacing-4: 16px;
}

[data-theme="dark"] {
  --color-primary: #60a5fa;
  --color-background: #111827;
  --color-text: #f9fafb;
}
```

## Layer 2: Component Primitives

Headless/unstyled base components providing accessibility and behavior without visual opinions. Libraries: Radix UI, Headless UI, React Aria.

```tsx
// Button primitive — behavior and accessibility only
export function ButtonPrimitive({ children, disabled, onClick, ...props }) {
  return (
    <button
      disabled={disabled}
      onClick={onClick}
      type="button"
      {...props}
    >
      {children}
    </button>
  );
}

// Styled button — applies design tokens to the primitive
export const Button = styled(ButtonPrimitive)`
  background: var(--color-primary);
  color: white;
  padding: var(--spacing-2) var(--spacing-4);
  border-radius: 6px;
  font-size: var(--font-size-sm);
  &:hover:not(:disabled) { background: var(--color-primary-hover); }
  &:disabled { opacity: 0.5; cursor: not-allowed; }
`;
```

## Layer 3: Component Variants with CVA

```tsx
import { cva, type VariantProps } from 'class-variance-authority';

const buttonVariants = cva(
  // Base classes
  'inline-flex items-center justify-center rounded-md font-medium transition-colors',
  {
    variants: {
      variant: {
        primary: 'bg-blue-600 text-white hover:bg-blue-700',
        secondary: 'bg-gray-100 text-gray-900 hover:bg-gray-200',
        ghost: 'hover:bg-gray-100',
        destructive: 'bg-red-600 text-white hover:bg-red-700',
      },
      size: {
        sm: 'h-8 px-3 text-sm',
        md: 'h-10 px-4',
        lg: 'h-12 px-6 text-lg',
      },
    },
    defaultVariants: {
      variant: 'primary',
      size: 'md',
    },
  }
);

export interface ButtonProps
  extends React.ButtonHTMLAttributes<HTMLButtonElement>,
    VariantProps<typeof buttonVariants> {}

export const Button = ({ variant, size, className, ...props }: ButtonProps) => (
  <button className={cn(buttonVariants({ variant, size }), className)} {...props} />
);
```

## Atomic Design Hierarchy

```
Tokens
  └── Atoms (Button, Input, Icon, Badge, Avatar)
        └── Molecules (SearchInput, FormField, Card)
              └── Organisms (NavigationBar, DataTable, LoginForm)
                    └── Templates (PageLayout, DashboardLayout)
                          └── Pages (specific app instances)
```

## Documentation with Storybook

```tsx
// Button.stories.tsx

const meta: Meta<typeof Button> = {
  title: 'Components/Button',
  component: Button,
  argTypes: {
    variant: { control: 'select', options: ['primary', 'secondary', 'ghost'] },
    size: { control: 'radio', options: ['sm', 'md', 'lg'] },
  },
};
export default meta;

export const Primary: StoryObj<typeof Button> = {
  args: { variant: 'primary', children: 'Click me' },
};
```

## Versioning and Distribution

- **Semantic versioning**: patch for bug fixes, minor for new components, major for breaking changes
- **Changesets**: document changes, auto-generate changelogs, coordinate multi-package releases
- **Peer dependencies**: React as `peerDependency` to avoid bundling multiple React versions

---

## Interview Discussion Points

**Q: How do you handle breaking changes in a design system?**
Deprecation warnings in v+0.x, removal in next major. Keep old API aliases during transition. Document migration guides in changelog.

**Q: How do you enforce token usage vs raw values?**
ESLint rules (eslint-plugin-no-restricted-syntax to ban raw color hex values), design system linting (Stylelint), Figma variables linked to code tokens.

**Q: What makes a component "accessible by default"?**
Using semantic HTML, Radix UI primitives (handle ARIA, keyboard nav, focus management), testing with axe-core in CI, and including ARIA patterns in Storybook stories.
