# Container/Presentational Components Pattern

## Overview

The Container/Presentational pattern (also called Smart/Dumb or Stateful/Stateless) separates components into two categories:

- **Container (Smart)**: Handles logic, state, data fetching
- **Presentational (Dumb)**: Handles UI, receives data via props

This pattern promotes separation of concerns and reusability.

## The Pattern

### Basic Structure

```jsx
// Presentational Component: Just UI
function UserCard({ user, onEdit, onDelete }) {
  return (
    <div className="user-card">
      <img src={user.avatar} alt={user.name} />
      <h2>{user.name}</h2>
      <p>{user.email}</p>
      <button onClick={onEdit}>Edit</button>
      <button onClick={onDelete}>Delete</button>
    </div>
  );
}

// Container Component: Logic and state
function UserCardContainer({ userId }) {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);
  
  useEffect(() => {
    fetchUser(userId).then(data => {
      setUser(data);
      setLoading(false);
    });
  }, [userId]);
  
  const handleEdit = () => {
    // Edit logic
    navigate(`/users/${userId}/edit`);
  };
  
  const handleDelete = async () => {
    await deleteUser(userId);
    navigate('/users');
  };
  
  if (loading) return <Spinner />;
  
  return (
    <UserCard
      user={user}
      onEdit={handleEdit}
      onDelete={handleDelete}
    />
  );
}
```

## Characteristics

### Presentational Components

**Characteristics:**
- Concerned with **how things look**
- Receive data and callbacks via props
- Rarely have own state (only UI state)
- Written as function components
- Highly reusable
- Easy to test

```jsx
// Purely presentational
function Button({ label, onClick, variant = 'primary', disabled }) {
  return (
    <button
      className={`btn btn-${variant}`}
      onClick={onClick}
      disabled={disabled}
    >
      {label}
    </button>
  );
}

function ProductCard({ product, onAddToCart, onViewDetails }) {
  return (
    <div className="product-card">
      <img src={product.image} alt={product.name} />
      <h3>{product.name}</h3>
      <p>${product.price}</p>
      <Button label="Add to Cart" onClick={() => onAddToCart(product)} />
      <Button 
        label="View Details" 
        onClick={() => onViewDetails(product)}
        variant="secondary"
      />
    </div>
  );
}
```

### Container Components

**Characteristics:**
- Concerned with **how things work**
- Provide data and behavior to presentational components
- Often stateful
- Call Redux/API/services
- Generated by higher-order components or hooks

```jsx
// Container with logic
function ProductCardContainer({ productId }) {
  const [product, setProduct] = useState(null);
  const { addToCart } = useCart();
  const navigate = useNavigate();
  
  useEffect(() => {
    fetchProduct(productId).then(setProduct);
  }, [productId]);
  
  const handleAddToCart = (product) => {
    addToCart(product);
    toast.success('Added to cart!');
  };
  
  const handleViewDetails = (product) => {
    navigate(`/products/${product.id}`);
  };
  
  if (!product) return <Skeleton />;
  
  return (
    <ProductCard
      product={product}
      onAddToCart={handleAddToCart}
      onViewDetails={handleViewDetails}
    />
  );
}
```

## Practical Examples

### Example 1: Todo List

```jsx
// Presentational: Todo UI
function TodoItem({ todo, onToggle, onDelete }) {
  return (
    <div className={`todo-item ${todo.completed ? 'completed' : ''}`}>
      <input
        type="checkbox"
        checked={todo.completed}
        onChange={() => onToggle(todo.id)}
      />
      <span>{todo.text}</span>
      <button onClick={() => onDelete(todo.id)}>Delete</button>
    </div>
  );
}

function TodoList({ todos, onToggle, onDelete }) {
  return (
    <div className="todo-list">
      {todos.map(todo => (
        <TodoItem
          key={todo.id}
          todo={todo}
          onToggle={onToggle}
          onDelete={onDelete}
        />
      ))}
    </div>
  );
}

// Container: Todo logic
function TodoListContainer() {
  const [todos, setTodos] = useState([]);
  const [loading, setLoading] = useState(true);
  
  useEffect(() => {
    fetchTodos().then(data => {
      setTodos(data);
      setLoading(false);
    });
  }, []);
  
  const handleToggle = async (id) => {
    // Optimistic update
    setTodos(prev =>
      prev.map(todo =>
        todo.id === id ? { ...todo, completed: !todo.completed } : todo
      )
    );
    
    // API call
    await toggleTodo(id);
  };
  
  const handleDelete = async (id) => {
    setTodos(prev => prev.filter(todo => todo.id !== id));
    await deleteTodo(id);
  };
  
  if (loading) return <Spinner />;
  
  return (
    <TodoList
      todos={todos}
      onToggle={handleToggle}
      onDelete={handleDelete}
    />
  );
}
```

### Example 2: User Profile

```jsx
// Presentational: Profile display
function ProfileInfo({ user }) {
  return (
    <div className="profile-info">
      <h1>{user.name}</h1>
      <p>{user.email}</p>
      <p>{user.bio}</p>
    </div>
  );
}

function ProfileStats({ stats }) {
  return (
    <div className="profile-stats">
      <div>Posts: {stats.posts}</div>
      <div>Followers: {stats.followers}</div>
      <div>Following: {stats.following}</div>
    </div>
  );
}

function ProfileActions({ onEdit, onFollow, isFollowing }) {
  return (
    <div className="profile-actions">
      <button onClick={onEdit}>Edit Profile</button>
      <button onClick={onFollow}>
        {isFollowing ? 'Unfollow' : 'Follow'}
      </button>
    </div>
  );
}

// Container: Profile logic
function ProfileContainer({ userId }) {
  const { currentUser } = useAuth();
  const [user, setUser] = useState(null);
  const [stats, setStats] = useState(null);
  const [isFollowing, setIsFollowing] = useState(false);
  
  useEffect(() => {
    Promise.all([
      fetchUser(userId),
      fetchUserStats(userId),
      checkFollowing(userId)
    ]).then(([userData, statsData, followingStatus]) => {
      setUser(userData);
      setStats(statsData);
      setIsFollowing(followingStatus);
    });
  }, [userId]);
  
  const handleEdit = () => {
    navigate(`/profile/edit`);
  };
  
  const handleFollow = async () => {
    if (isFollowing) {
      await unfollowUser(userId);
      setIsFollowing(false);
      setStats(prev => ({ ...prev, followers: prev.followers - 1 }));
    } else {
      await followUser(userId);
      setIsFollowing(true);
      setStats(prev => ({ ...prev, followers: prev.followers + 1 }));
    }
  };
  
  if (!user || !stats) return <Skeleton />;
  
  const isOwnProfile = currentUser.id === userId;
  
  return (
    <>
      <ProfileInfo user={user} />
      <ProfileStats stats={stats} />
      {isOwnProfile && <ProfileActions onEdit={handleEdit} />}
      {!isOwnProfile && (
        <ProfileActions
          onFollow={handleFollow}
          isFollowing={isFollowing}
        />
      )}
    </>
  );
}
```

### Example 3: Comment Section

```jsx
// Presentational components
function Comment({ comment, onReply, onLike, onDelete, canDelete }) {
  return (
    <div className="comment">
      <div className="comment-header">
        <img src={comment.author.avatar} alt={comment.author.name} />
        <span>{comment.author.name}</span>
        <span>{formatDate(comment.createdAt)}</span>
      </div>
      
      <p className="comment-body">{comment.text}</p>
      
      <div className="comment-actions">
        <button onClick={() => onLike(comment.id)}>
          ❤️ {comment.likes}
        </button>
        <button onClick={() => onReply(comment.id)}>
          Reply
        </button>
        {canDelete && (
          <button onClick={() => onDelete(comment.id)}>
            Delete
          </button>
        )}
      </div>
    </div>
  );
}

function CommentList({ comments, onReply, onLike, onDelete, currentUserId }) {
  return (
    <div className="comment-list">
      {comments.map(comment => (
        <Comment
          key={comment.id}
          comment={comment}
          onReply={onReply}
          onLike={onLike}
          onDelete={onDelete}
          canDelete={comment.author.id === currentUserId}
        />
      ))}
    </div>
  );
}

// Container component
function CommentSectionContainer({ postId }) {
  const { currentUser } = useAuth();
  const [comments, setComments] = useState([]);
  const [replyTo, setReplyTo] = useState(null);
  
  useEffect(() => {
    fetchComments(postId).then(setComments);
  }, [postId]);
  
  const handleReply = (commentId) => {
    setReplyTo(commentId);
  };
  
  const handleLike = async (commentId) => {
    // Optimistic update
    setComments(prev =>
      prev.map(c =>
        c.id === commentId ? { ...c, likes: c.likes + 1 } : c
      )
    );
    
    await likeComment(commentId);
  };
  
  const handleDelete = async (commentId) => {
    if (!confirm('Delete comment?')) return;
    
    setComments(prev => prev.filter(c => c.id !== commentId));
    await deleteComment(commentId);
  };
  
  const handleSubmitReply = async (text) => {
    const newComment = await createComment(postId, text, replyTo);
    setComments(prev => [...prev, newComment]);
    setReplyTo(null);
  };
  
  return (
    <>
      <CommentList
        comments={comments}
        onReply={handleReply}
        onLike={handleLike}
        onDelete={handleDelete}
        currentUserId={currentUser.id}
      />
      
      {replyTo && (
        <ReplyForm
          onSubmit={handleSubmitReply}
          onCancel={() => setReplyTo(null)}
        />
      )}
    </>
  );
}
```

### Example 4: Data Table

```jsx
// Presentational: Table UI
function Table({ columns, data, onSort, sortBy, sortOrder }) {
  return (
    <table>
      <thead>
        <tr>
          {columns.map(col => (
            <th
              key={col.key}
              onClick={() => onSort(col.key)}
              style={{ cursor: 'pointer' }}
            >
              {col.label}
              {sortBy === col.key && (sortOrder === 'asc' ? ' ↑' : ' ↓')}
            </th>
          ))}
        </tr>
      </thead>
      <tbody>
        {data.map((row, i) => (
          <tr key={i}>
            {columns.map(col => (
              <td key={col.key}>
                {col.render ? col.render(row[col.key], row) : row[col.key]}
              </td>
            ))}
          </tr>
        ))}
      </tbody>
    </table>
  );
}

function Pagination({ currentPage, totalPages, onPageChange }) {
  return (
    <div className="pagination">
      <button
        onClick={() => onPageChange(currentPage - 1)}
        disabled={currentPage === 1}
      >
        Previous
      </button>
      
      <span>
        Page {currentPage} of {totalPages}
      </span>
      
      <button
        onClick={() => onPageChange(currentPage + 1)}
        disabled={currentPage === totalPages}
      >
        Next
      </button>
    </div>
  );
}

// Container: Table logic
function UserTableContainer() {
  const [users, setUsers] = useState([]);
  const [page, setPage] = useState(1);
  const [sortBy, setSortBy] = useState('name');
  const [sortOrder, setSortOrder] = useState('asc');
  const [totalPages, setTotalPages] = useState(1);
  
  useEffect(() => {
    fetchUsers({ page, sortBy, sortOrder }).then(data => {
      setUsers(data.users);
      setTotalPages(data.totalPages);
    });
  }, [page, sortBy, sortOrder]);
  
  const handleSort = (key) => {
    if (sortBy === key) {
      setSortOrder(order => order === 'asc' ? 'desc' : 'asc');
    } else {
      setSortBy(key);
      setSortOrder('asc');
    }
  };
  
  const columns = [
    { key: 'name', label: 'Name' },
    { key: 'email', label: 'Email' },
    {
      key: 'actions',
      label: 'Actions',
      render: (_, user) => (
        <button onClick={() => handleEdit(user.id)}>Edit</button>
      )
    }
  ];
  
  return (
    <>
      <Table
        columns={columns}
        data={users}
        onSort={handleSort}
        sortBy={sortBy}
        sortOrder={sortOrder}
      />
      <Pagination
        currentPage={page}
        totalPages={totalPages}
        onPageChange={setPage}
      />
    </>
  );
}
```

## Modern Alternatives

### With Custom Hooks

```jsx
// Extract logic into custom hook
function useUserProfile(userId) {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);
  
  useEffect(() => {
    fetchUser(userId).then(data => {
      setUser(data);
      setLoading(false);
    });
  }, [userId]);
  
  const updateUser = async (updates) => {
    await updateUserAPI(userId, updates);
    setUser(prev => ({ ...prev, ...updates }));
  };
  
  return { user, loading, updateUser };
}

// Component becomes simpler
function UserProfile({ userId }) {
  const { user, loading, updateUser } = useUserProfile(userId);
  
  if (loading) return <Spinner />;
  
  return (
    <div>
      <h1>{user.name}</h1>
      <p>{user.email}</p>
      <button onClick={() => updateUser({ name: 'New Name' })}>
        Update
      </button>
    </div>
  );
}
```

## Advantages

1. **Separation of Concerns**: Logic separate from UI
2. **Reusability**: Presentational components easily reused
3. **Testability**: Test UI and logic independently
4. **Maintainability**: Easy to find and modify behavior
5. **Designer-Friendly**: Designers can work on presentational components

## Disadvantages

1. **Boilerplate**: More files and components
2. **Prop Drilling**: Can lead to many props passed down
3. **Abstraction**: Sometimes over-engineers simple components
4. **Outdated**: Custom hooks often better for logic reuse

## When to Use

### Use Container/Presentational When:

- Building a component library
- Need to reuse UI with different data sources
- Team has designers working on UI separately
- Complex state management needs isolation

### Use Custom Hooks Instead When:

- Logic is the primary concern
- No need to share UI between different logic
- Want simpler component structure
- Building modern React apps

## Testing

### Testing Presentational Components

```jsx
import { render, screen } from '@testing-library/react';

test('UserCard displays user info', () => {
  const user = {
    name: 'John Doe',
    email: 'john@example.com',
    avatar: '/avatar.jpg'
  };
  
  render(<UserCard user={user} onEdit={() => {}} onDelete={() => {}} />);
  
  expect(screen.getByText('John Doe')).toBeInTheDocument();
  expect(screen.getByText('john@example.com')).toBeInTheDocument();
});

test('UserCard calls callbacks', () => {
  const onEdit = jest.fn();
  const onDelete = jest.fn();
  
  render(
    <UserCard
      user={{ name: 'John', email: 'john@example.com' }}
      onEdit={onEdit}
      onDelete={onDelete}
    />
  );
  
  fireEvent.click(screen.getByText('Edit'));
  expect(onEdit).toHaveBeenCalled();
  
  fireEvent.click(screen.getByText('Delete'));
  expect(onDelete).toHaveBeenCalled();
});
```

### Testing Container Components
